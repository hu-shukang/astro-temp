## 複数契約（マルチホーム）

1ユーザーが複数の住所（自宅・別荘・投資物件など）に電力契約を持てる。

### 契約切り替え（ContractSelector）

電気使用量・電気代・支払い方法などの参照・操作ページには、ページ上部に**契約切り替えセレクター**を表示する。

```
[契約切り替え ▼]
  ✓ 渋谷区神南1-1-1（スタンダード 30A）      ← 現在表示中
    横浜市中区山下町1-1（エコプラン 20A）
    + 新規契約を申し込む
```

- 表示内容：**供電住所（都道府県＋市区町村＋番地）+ プラン名 + アンペア**
- 選択中の契約は `contractStore.selectedContractId` で全ページ共有
- 選択変更時：即座にデータを再取得・再描画（ローディング表示あり）
- 契約が1件しかない場合：セレクターは表示するが切り替えは不要（UX上は表示したまま）

### ContractSelectorを表示するページ

- `/electricity/monthly`
- `/electricity/daily`
- `/billing`（一覧・詳細）
- `/account/payment`（支払い方法変更）
- `/service/change-plan`・`/service/terminate`（操作対象契約の選択）

### ダッシュボード（`/`）での扱い

- ダッシュボードは**全契約のサマリー**を一覧表示する
- 個別確認は各ページのContractSelectorで行う

## 支払い方法

| 種別             | 詳細                           | 備考                                      |
| ---------------- | ------------------------------ | ----------------------------------------- |
| クレジットカード | Visa / Mastercard / JCB / Amex | カード番号・有効期限・名義を登録          |
| 口座振替         | 銀行口座から自動引落し         | 金融機関・口座番号・名義を登録            |
| コンビニ払込票   | 毎月払込票を郵送・発行         | 手数料あり（¥110/回）、事前に申し込み不要 |

- 契約ごとに支払い方法を設定可能
- 変更は翌月請求分から反映
- コンビニ払込票は払込期限（請求月の翌月末）を超えると延滞扱い

## 退会の特殊ルール

```
退会申請フロー：
1. 有効な契約が存在する場合
   → 「全ての契約を解約してから退会できます」と警告
   → /service/terminate へ誘導
2. 有効な契約が0件の場合
   → パスワード確認入力を表示
   → 退会処理実行
```

## お知らせ

### 通知の種類

| 種別              | トリガー                     | 例                                                |
| ----------------- | ---------------------------- | ------------------------------------------------- |
| `bill_issued`     | 請求書発行時                 | 「2025年3月分の請求書が発行されました（¥8,420）」 |
| `bill_due`        | 支払期限3日前                | 「お支払期限が3日後に迫っています（¥8,420）」     |
| `usage_alert`     | 月間使用量が前月比120%超過時 | 「今月の使用量が先月より20%多くなっています」     |
| `contract_update` | 契約変更・申請完了時         | 「プラン変更の申請が完了しました」                |
| `points_expiring` | ポイント失効30日前           | 「1,200ポイントが30日後に失効します」             |
| `campaign`        | キャンペーン告知             | 「夏の節電キャンペーン開始のお知らせ」            |
| `system`          | システムメンテナンスなど     | 「4月1日 2:00〜4:00 メンテナンスのお知らせ」      |

### 機能仕様

- 一覧表示：新着順、未読件数をサイドバー・ナビバーにバッジ表示
- 既読管理：個別既読・一括既読
- 詳細表示：お知らせをクリックで詳細内容を表示（モーダルまたは展開）
- 関連リンク：請求書お知らせ → 該当請求書詳細へ遷移、など

| ---------------- | ---------------------------------------- |
| 料金・請求       | 「電気料金はいつ引き落とされますか？」   |

- 「この回答は役に立ちましたか？」フィードバックボタン（任意）
- ページ下部に「解決しない場合はお問い合わせへ」リンク

## お問い合わせ

### 問い合わせ種別

| 種別                       | 説明                         |
| -------------------------- | ---------------------------- |
| 料金・請求に関するご質問   | 請求書・支払いについて       |
| 契約・プランに関するご質問 | 申込・変更・解約について     |
| 電力使用量に関するご質問   | データ・計器について         |
| ポイントに関するご質問     | 付与・使用・交換について     |
| Webサービスに関するご質問  | ログイン・アカウントについて |
| その他                     | 上記以外                     |

### 機能仕様

- **送信フォーム**（`/support/contact`）：種別選択・件名・本文・添付ファイル（任意）
- **問い合わせ履歴**（`/support/contact/history`）：送信済み一覧・ステータス（受付中・回答済み）
- 送信後：確認番号を発行し完了画面に表示
- 回答が来たらお知らせ（`contact_replied`）で通知

## ポイントシステム

| 項目           | 内容                                     |
| -------------- | ---------------------------------------- |
| 付与タイミング | 毎月（契約継続ボーナス）・キャンペーン   |
| 使用方法       | 電気代充当（1P = 1円）／他社ポイント交換 |
| 有効期限       | 付与から2年（失効前に通知）              |
| 交換先         | Tポイント・楽天ポイント・dポイント・WAON |

## ステータス一覧

### 契約ステータス

- `active` : 有効
- `pending` : 申請中
- `terminated` : 解約済み
- `transferring` : 乗り換え手続き中

### 請求書ステータス

- `unpaid` : 未払い
- `paid` : 支払済み
- `overdue` : 期限超過
- `processing` : 処理中
